# CLAUDE.md — LLM Wiki 운영 규칙

이 저장소는 **LLM이 직접 작성·유지하는 영속적 마크다운 위키**다.
전체 컨셉과 아키텍처는 [PLAN.md](./PLAN.md) 참조.

이 문서는 LLM(Claude Code)이 이 저장소에서 작업할 때 따라야 할 **운영 규칙**이다.

---

## 1. 3계층 원칙 (절대 위반 금지)

- `sources/` 는 **읽기 전용**. 절대 수정·삭제하지 않는다.
- `wiki/` 는 LLM이 작성·갱신한다.
- `CLAUDE.md` / `PLAN.md` 는 사람이 정의하는 운영 규칙. LLM은 사람이 명시적으로 요청한 경우에만 수정한다.

## 2. 언어 정책

- 위키 본문은 **한국어**로 작성한다.
- 인용·고유명사·기술 용어는 **원문 표기를 보존**한다 (예: "transformer", "RAG", "self-attention").
- 한국어 번역어가 통용되면 첫 등장 시 "원어(번역)" 또는 "번역(원어)" 형태로 병기.

## 3. 페이지 작성 규칙

모든 위키 페이지 상단에 frontmatter:

```yaml
---
title: <페이지 제목>
type: concept | entity | topic | summary
sources: [<원문 ID>, ...]
updated: YYYY-MM-DD
---
```

본문 규칙:
- 모든 사실 진술 끝에 `[^source-id]` 출처 표시
- 다른 페이지 언급 시 `[페이지명](../concepts/page.md)` 마크다운 링크
- 추측·해석은 명시적으로 `추론:` 으로 시작
- 날짜는 항상 절대 표기 (YYYY-MM-DD), "어제/지난주" 금지

## 4. 워크플로

### 4.1 Ingest (수집)
사용자가 새 원문을 `sources/` 에 넣고 인제스트를 요청하면:

1. 원문을 읽고 `wiki/summaries/<slug>.md` 요약 페이지 생성
2. 추출된 개념/개체에 대해 `wiki/concepts/`, `wiki/entities/` 의 **기존 페이지 갱신**
   - 기존에 없던 것만 신규 생성
   - 한 번의 인제스트에서 10~15개 페이지가 함께 갱신되는 게 정상
3. 새/갱신된 페이지에 교차링크 추가
4. `wiki/index.md` 에 신규 페이지 등록
5. `wiki/log.md` 에 변경 내역 기록 (날짜 + 변경된 파일 목록 + 한 줄 요약)
6. `sources/INDEX.md` 에 원문 메타데이터(ID/제목/URL/날짜) 추가

### 4.2 Query (질의)
사용자가 위키 기반으로 질문하면:

1. `wiki/index.md` 에서 관련 페이지 후보 식별
2. 해당 페이지들을 읽고 답변 합성, 필요시 원문 인용
3. 답변이 재사용 가치 있다고 판단되면 `wiki/topics/` 에 페이지로 승격 제안 (사용자 승인 후 작성)

### 4.3 Lint (정합성 점검)
사용자가 lint를 요청하면 다음을 점검:

- 모순되는 진술 (같은 사실에 대한 상충 기록)
- 고아 페이지 (어디서도 링크되지 않는 페이지)
- 누락된 교차참조 (페이지 A가 B를 언급하지만 링크 없음)
- 노후 페이지 (원문 갱신 후 위키 미반영)

결과는 보고만 하고, 자동 수정은 사용자 승인 후 진행.

## 5. 금지 사항

- `sources/` 내용 수정·이동·삭제
- 출처 없는 사실 진술 (모든 fact는 `[^source-id]` 또는 "추론:" 표시)
- 페이지 삭제는 사용자 명시 승인 없이 금지 (대신 `deprecated: true` frontmatter 추가)
- 원문에 없는 정보를 사실처럼 단언 (확장 추론은 반드시 "추론:" 표시)

## 6. 커밋 정책

- ingest/update/lint 단위로 커밋
- 커밋 메시지 형식: `<workflow>: <한 줄 요약>` (예: `ingest: karpathy LLM wiki gist`)
- 한 번의 인제스트는 한 번의 커밋으로 묶는다 (관련 변경 추적 용이)
